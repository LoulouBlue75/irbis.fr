# Templates - Mode Compound

## Philosophie

Les templates Irbis ne sont pas des documents figes. Ils evoluent selon le principe **compound** :
- Chaque mission enrichit le corpus
- Les patterns qui marchent sont capitalises
- Les frictions sont eliminees iterativement

---

## Boucle d'Amelioration

```
MISSION N
    ↓
Utilisation template
    ↓
Friction / Insight identifie
    ↓
LEARNING LOG (ci-dessous)
    ↓
Template MAJ
    ↓
MISSION N+1 (amelioree)
```

## Regles Compound

1. **Jamais supprimer** - deprecier et archiver
2. **Toujours documenter** - le "pourquoi" du changement
3. **Tester avant generaliser** - valider sur 2-3 missions
4. **Small increments** - pas de refonte massive
