# Templates - Mode Compound

## Philosophie

Les templates Irbis ne sont pas des documents figes. Ils evoluent selon le principe **compound** :
- Chaque mission enrichit le corpus
- Les patterns qui marchent sont capitalises
- Les frictions sont eliminees iterativement

---


## Dimensionnement Orchestrator

Le corpus compound accumule une **densite operatoire maximale**. L'orchestrator extrait le **pourcentage pertinent** selon le contexte.

### Facteurs de Dimensionnement

| Facteur | Impact | Exemple |
|---------|--------|---------|
| **Seniority poste** | Profondeur cadrage | C-Level = 100%, Manager = 60% |
| **Complexite mission** | Nb sections actives | Multi-pays = full, Local = light |
| **Maturite client** | Detail contractuel | Nouveau = premium, Recurrent = standard |
| **Urgence** | Completude evaluation | Urgent = scorecard only, Normal = full |
| **Fee structure** | Formalisme | Retainer = complet, Success = essentiel |

### Logique de Selection

```
DISCOVERY (orchestrator + user)
      ↓
Questions contextuelles :
- Quel poste ? Quel niveau ?
- Quel secteur ? Quelle geographie ?
- Client nouveau ou existant ?
- Urgence ? Contraintes specifiques ?
- Retainer ou success only ?
      ↓
CONTEXTE COMPRIS
      ↓
DIMENSIONNEMENT
- Cadrage : sections 1-8 ou subset
- Evaluation : full process ou accelere
- Contract : preset + ajustements
      ↓
OUTPUT CONTEXTUALISE
```

### Discovery First

> L'orchestrator ne presume jamais.
> Il mene la discovery necessaire avec l'user.
> Le dimensionnement decoule du contexte decouvert.

**Modes de discovery** :
- **Explicite** : questions directes a l'user
- **Implicite** : inference depuis documents fournis (brief client, emails, notes)
- **Iterative** : affinement au fil de la mission

**Anti-pattern** : dimensionner avant d'avoir compris → output hors sujet

### Exemples Dimensionnement

**Mission Premium C-Level International**
- Cadrage : 100% (8/8 sections, 15+ entreprises cibles)
- Evaluation : Full process 4 etapes + assessment externe
- Contract : Preset premium + clauses specifiques

**Mission Manager Local Recurrent**
- Cadrage : 50% (sections 1, 3, 5, 6, 8)
- Evaluation : Scorecard + entretien (2 etapes)
- Contract : Preset standard, signature rapide

**Mission Urgente Success-Only**
- Cadrage : 30% (sections 1, 6, 8 essentielles)
- Evaluation : Scorecard uniquement
- Contract : Preset success_only

### Principe Cle

> La richesse du corpus n'impose pas sa completude.
> L'orchestrator dimensionne intelligemment.
> Moins mais mieux > tout mais dilue.

---

## Regles Compound

1. **Jamais supprimer** - deprecier et archiver
2. **Toujours documenter** - le "pourquoi" du changement
3. **Tester avant generaliser** - valider sur 2-3 missions
4. **Small increments** - pas de refonte massive
